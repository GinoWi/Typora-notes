# 网页布局方式

什么是网页布局方式？

网页布局方式其实就是指浏览器是如何对网页中的元素进行排版的。



### 标准的排版方式

又叫做（文档流/普通流）排版方式

+ 其实浏览器默认的排版方式就是标准流排版方式。

+ 在CSS中将元素分为三类，分别是块级元素/行内元素/行内块级元素。

+ 在标准流中有两种排版方式，一种是垂直排版，一种是水平排版。如果元素是块级元素就会垂直排版，如果是行内/行内块级元素



### 浮动流排版方式

+ 浮动流是一种“半脱离标准流”的排版方式
+ 浮动流只有一种排版方式，就是水平排版。它只能设置某个元素左对齐或者右对齐。
+ 注意点：
  + 浮动流中没有居中对齐，也就是没有center这个取值
  + 在浮动流中不可以使用margin:0 auto;的
+ 特点：
  + 在浮动流中是不区分块级元素/行内元素/行内块级元素，无论是块级元素/行内元素/行内块级元素都可以水平排版。
  + 在浮动流中无论是块级元素/行内元素/行内块级元素都可以设置宽高。
  + 综上所述，浮动流中的元素和标准流中的行内块级元素很像。



#### 浮动元素脱标

脱标：脱离标准流

什么是浮动元素脱标？

当某一个元素浮动之后，那么这个元素看上去就像是被从标准流中删除了一样，这个就是浮动元素的脱标。



浮动元素脱标之后会有什么影响？

如果前面一个元素浮动了，而后面一个元素没有浮动，那么这个时候前面的一个元素就会盖住后面的一个元素。



#### 浮动元素排序规则

+ 相同方向上的浮动元素，先浮动的元素会显示在前面，后浮动的元素会显示在后面
+ 不同方向上的浮动元素，左浮动会找左浮动，右浮动会找右浮动
+ 浮动元素浮动之后的位置，由浮动元素浮动之前在标准流中的位置来确定



#### 浮动元素的贴靠现象

当父元素的宽度足够放下所有浮动元素的时候就会并列显示，如果不够放下所有浮动元素的时候，会从最后一个浮动元素开始一次找钱一个浮动元素贴靠，直到找到父元素贴靠，此时不管宽度是否能够放下，不再变化。

浮动会尽可能保证浮动元素在父元素中完整显示



#### 浮动元素字围现象

浮动元素不会盖住没有浮动的元素的文字，没有浮动元素的文字，会给浮动元素让位置。



#### 浮动元素高度问题

+ 在标准流中，内容的高度是可以撑起父元素的高度的
+ 在浮动流中，浮动的元素是无法撑起父元素的高度的



#### 清除浮动方式

方式一：

+ 给前面的父元素设置高度
+ 注意点：
  + 在企业开发中能不写高度就不写高度，所以这种方式使用很少。

方式二：

+ 给后面的盒子添加clear属性
+ clear属性取值：
  + none：默认取值，按照浮动元素的排序规则来排序（左浮动找左浮动，右浮动找右浮动）
  + left：不要找前面的左浮动元素
  + right：不要找前面的右浮动元素
  + both：不要找前面的左浮动元素和右浮动元素
+ 注意点：
  + 当给某个元素添加clear属性，那么这个属性的margin属性就会失效。为什么margin属性会失效，原因就是给首个子元素（若前面的盒子元素高度为0表示盒子不存在）设置margin属性，会将父元素一起顶下来。

方式三：

+ 隔墙法（需要额外添加块级元素）

  + 外墙法
    + 在两个盒子中间额外添加块级元素
    + 给这个额外添加的块级元素添加clear:both属性
    + 注意点：
      + 外墙法它可以让第二个盒子使用margin-top属性
      + 外墙法不可以让第一个盒子使用margin-bottom属性
      + 一般不给两个盒子设置margin属性，而是通过额外的盒子设置高度属性来实现margin效果
  + 内墙法
    + 在第一个盒子中所有子元素最后添加额外块级元素
    + 给这个额外添加的块级元素添加clear:both;属性
    + 注意点：
      + 外墙法它可以让第二个盒子使用margin-top属性
      + 内墙法可以让第一个盒子使用margin-bottom属性
      + 一般不给两个盒子设置margin属性，而是通过额外的盒子设置高度属性来实现margin效果
  + 两者区别
    + 外墙法不能撑起第一个盒子的高度，内墙法可以撑起第一个盒子的高度
  + 在开发中，一般不常用隔墙法清除浮动。
