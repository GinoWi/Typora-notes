# 盒子模型

### 什么是CSS的盒子模型？

CSS盒子模型仅仅是一个形象的比喻，HTML中所有标签都是一个盒子

结论：

1. 在HTML中所有标签都可以设置
   + 宽度、高度：指定可以存放内容的区域
   + 内边距
   + 外边距
   + 边框



### 盒子模型的宽度和高度

+ 内容的宽度和高度
  + 就是通过标签的width/height属性设置的宽度和高度。
+ 元素的宽度和高度
  + 就是当前这个标签能够看到的宽度和高度。
  + 宽度 = 左边框 + 左内边距 + 内容宽度 + 右内边距 + 右边框
  + 高度 = 上边框 + 上内边距 + 内容高度 + 下内边距 + 下边框
+ 元素空间的宽度和高度
  + 宽度 = 左外边距 + 左边框 + 左内边距 + 内容宽度 + 右内边距 + 右边框 + 右外边距
  + 高度 = 上外边距 + 上边框 + 上内边距 + 内容高度 + 下内边距 + 下边框 + 下外边距

+ 规律：
  + 增加了padding之后元素的宽高也会发生变化。
  + 如果增加了padding之后还想保持元素的宽高，那么就必须减去内容的宽高。
  + 增加了border之后元素的宽高也会发生变化。
  + 如果增加了border之后还想保持元素的宽高，那么就必须减去内容的宽高。
  + 如果两个盒子是嵌套关系，如果设置了里面和子顶部外边距，外面的盒子也会被顶下来。如果外面的盒子不想一起被顶下来，可以给外面的盒子设置一个边框属性。
  + 在开发过程中，一般情况下，需要控制嵌套关系盒子之间的距离，应该首先考虑padding，其次再考虑margin。marigin本质上是用于控制兄弟关系之间的间隙的。
  + 在嵌套关系的盒子中，可以通过margin: 0 auto;的方式让里面的盒子在外面的盒子水平居中。margin:0 auto;只对水平方向有效对垂直方向没有效果。



### 盒子的box-sizing属性

CSS3中新增了一个box-sizing属性，这个属性保证了，我们给盒子新增border或者padding属性之后，盒子元素的宽度和高度不变。

box-sizing属性是如何保证增加border、padding属性之后，盒子元素的宽度和高度不变？

和我们之前实验的式样，增加padding、border属性之后，要想保证盒子元素的宽度和高度不变，那么就必须减去一部分内容的宽度和高度。

取值：

+ content-box：默认取值，元素宽高等于边框 + 内边距 + 内容宽高。
+ border-box：设置border和padding属性的宽度后，盒子元素宽高不变。元素宽高恒等于width属性。



### 盒子居中和内容居中

1. text-align:center;和margin:0 auto;的区别
   + text-align:center;作用：设置盒子中存储的文字和图片水平居中
   + margin:0 auto;作用：让盒子自己水平居中



### 清空默认边距

为什么要清空默认边距（外边距和内边距）？

在开发中，为了更好的控制盒子的宽高和计算盒子的宽高等等，所以在开发中，编写代码之前第一件事情就是清空默认的边距。

格式：

```
* {
	margin: 0;
	padding: 0;
}
```

注意点：

+ 通配符选择器会遍历当前界面中的所有标签，所以性能不好。
